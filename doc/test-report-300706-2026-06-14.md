## 三、实际数据获取路径分析

子 Agent 没有调用 MCP 的 `get_market_data`，而是自行用 Bash 构建了一个数据管道：

```
Bash: pip install mootdx                                    → 安装 A 股数据 SDK
Bash: PYTHONPATH=agent python3 -c "import..."               → 导入 V-T 内部库
Bash: curl -H "..." "http://web.ifzq.gtimg.cn/..."          → 腾讯财经 K 线 API
Bash: curl -v --connect-timeout 10 "https://push2.eastmoney.com/..."  → 东方财富实时行情
Bash: python3 -c "import akshare..."                         → 尝试 akshare 库
Bash: python3 -c "import numpy as np; ..."                   → 手写 EMA/ADX/RSI/BB/OBV 计算
```

## 六、根因分析

```
Prompt 中明确写了 "You MUST call get_market_data(...)"
                    ↓
子 Agent 没有调用 get_market_data
                    ↓
原因1: MCP 工具在子 agent 上下文中不可见或感知为不可用
原因2: Bash 是子 agent 最熟悉、最灵活的工具，惯性使用
原因3: 子 agent 认为 "只要能拿到数据就行"，不区分 MCP vs Bash 路径
原因4: Skill/Workflow 触发机制缺失 — agent 不会自动加载 vt-swarm-* 做多视角分析
```

## 八、测试结论

**🔴 Test 1 — 失败**

虽然子 Agent 正确避开了 `run_swarm` 这个已弃用工具，也产出了一份内容详实的分析报告，但它**完全绕过了 MCP 工具体系**，用 54 次 Bash 调用（curl/Python/pip）自行构建了一个数据管道。这违背了 Vibe-Trading → Claude Code 移植的核心设计理念：**通过 MCP 工具统一数据获取入口，确保数据源可管理、可追溯、可审计**。

关键改进方向：子 agent 需要明确知道 MCP 工具 `get_market_data` 是可用且首选的数据入口，Bash 只能用于补充计算，不能替代 MCP 数据管道。
